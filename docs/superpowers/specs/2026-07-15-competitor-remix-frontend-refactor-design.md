# Competitor Remix 前端重构设计

日期：2026-07-15

## 1. 目标

重构 `/competitor-remix.html` 的页面架构、状态管理和全部前端交互，同时保持现有 `video-content-ops` API 路径、请求字段、任务类型、状态查询、取消、重试和结果接口不变。

本次重构要解决以下问题：

- 后端任务类型直接暴露为用户功能，入口重复且差异难以理解。
- 三列节点画布把素材、参数和结果分散在不同步骤中，不适合持续操作。
- 单个脚本同时负责状态、DOM、校验、画布、上传和轮询，修改容易产生串扰。
- 切换任务会丢失上下文，运行中的任务会占用全局状态。
- 本地视频在完成 Base64 读取前没有及时反馈，上传或拖入后显得无响应。
- K 帧、手动 mask 和区域框选分别实现，坐标与画面状态容易不一致。

## 2. 范围

### 2.1 本次包含

- 将页面改成左侧功能导航、中间操作区、右侧任务状态与结果栏。
- 把 10 条现有执行路径合并为 5 个用户功能入口；底层为 9 种 `job_type`，其中 `ai_remove` 包含 auto/manual 两种模式。
- 使用原生 ES Modules 拆分页面职责，不引入构建工具或前端框架。
- 共享视频素材，并为每个功能模式独立保存草稿。
- 支持多个作业同时轮询，作业执行不锁住素材和参数编辑。
- 重做上传、拖放、视频预览、K 帧、手动 mask、区域框选和错误反馈。
- 增加请求构建、状态、坐标交互和页面关键路径测试。

### 2.2 本次不包含

- 修改 `video-content-ops` 服务。
- 修改 Aigc-platform 的 video-ops API 路径、请求或响应契约。
- 修改数据库结构、归档模型、鉴权方式或部署架构。
- 新增后端任务类型或承诺后端尚未开放的局部语义替换能力。
- 持久化本地视频文件或 Base64 内容到浏览器存储。

## 3. 用户功能模型

页面只展示 5 个用户功能。请求构建层将功能模式映射到现有 10 条执行路径。

| 用户功能 | 操作模式 | 底层请求 |
| --- | --- | --- |
| 去除画面元素 | 智能重绘 | `job_type=seedance_ai_remove` |
| 去除画面元素 | 自动检测 | `job_type=ai_remove`, `params.mode=auto` |
| 去除画面元素 | K 帧选对象 | `job_type=auto_ai_remove` |
| 去除画面元素 | 固定区域 | `job_type=ai_remove`, `params.mode=manual` |
| 局部遮挡与模糊 | 区域遮挡或区域模糊 | `job_type=mask_edit` |
| 局部遮挡与模糊 | 贴纸或水印模糊 | `job_type=sticker_blur` |
| 尾段处理 | 检测并裁剪导流尾段 | `job_type=end_trim_detection` |
| 语言处理 | 字幕翻译回写 | `job_type=video_copy_translate` |
| 语言处理 | 语言改写 | `job_type=language_rewrite` |
| 素材分析 | 分析报告文本 | `job_type=material_analysis` |

`job_type`、`mask_threshold`、`removal_engine` 等实现概念不作为一级功能名称。必要的技术参数放在“高级设置”中，作业详情可展示实际底层任务，便于排障。

## 4. 页面结构

### 4.1 左侧功能导航

左侧固定展示 5 个功能入口。每个入口显示：

- 用户可理解的功能名称和一句能力说明。
- 当前模式名称。
- 草稿、运行中、失败或已有结果状态。
- 是否需要选区或 K 帧交互。

左侧不展示底层任务卡片，不要求用户理解 provider 或 mask 类型。

### 4.2 中间操作区

中间区域由共享素材区和当前功能工作区组成。

共享素材区支持视频 URL、点击选择文件和拖放视频。选择本地文件后立即创建 Object URL，用于预览和读取元信息；此时不等待 Base64 转换。只有提交需要本地文件的作业时才开始读取 Base64，并展示准备进度。读取结果在当前页面生命周期内缓存，后续作业复用。

当前功能工作区使用分段控件选择模式，只显示该模式需要的参数和编辑器：

- 无选区模式显示基础参数。
- K 帧模式显示视频时间轴、当前帧、点选或框选工具和提示摘要。
- 固定区域模式直接在视频画面上框选，并填写作用时间段。
- 局部遮挡与模糊模式直接在视频画面上框选，并选择处理方式。
- 高级参数默认折叠，保留现有请求字段和默认值。
- 素材分析使用独立的报告文本输入，不依赖共享视频。

### 4.3 右侧作业栏

右侧固定展示当前提交条件和作业列表：

- 就绪清单逐项显示素材、参数和交互条件。
- 提交按钮只锁定当前提交过程，不锁定页面其他操作。
- 每次提交创建独立作业卡片，保存请求快照和归档入口。
- 作业卡片显示用户功能、模式、状态、队列、失败原因、结果和下载入口。
- 取消和重试只作用于指定作业。
- 重试继续调用现有作业 `/retry` 接口；页面使用原提交快照展示和确认重试上下文，不读取后来修改的草稿。

窄屏下左侧变为顶部功能切换器，右侧变为可展开的作业抽屉，中间工作区保持主操作位置。

## 5. 模块结构

页面继续使用无构建步骤的浏览器原生模块。

| 文件 | 职责 |
| --- | --- |
| `public/competitor-remix.html` | 平台外壳、三栏稳定容器和登录弹窗 |
| `public/competitor-remix.js` | 页面启动、模块装配和登录状态接入 |
| `public/competitor-remix/capability-catalog.js` | 5 个功能、模式、默认参数、编辑器和底层映射定义 |
| `public/competitor-remix/store.js` | 共享素材、模式草稿、作业集合和订阅机制 |
| `public/competitor-remix/payloads.js` | 纯函数校验、mask 生成输入和 API 请求构建 |
| `public/competitor-remix/media-workspace.js` | URL、文件、拖放、Object URL、视频元信息和延迟 Base64 |
| `public/competitor-remix/editors.js` | K 帧点选/框选、手动 mask、区域框选及坐标换算 |
| `public/competitor-remix/job-runner.js` | 提交、并行轮询、结果读取、取消、重试和恢复 |
| `public/competitor-remix/view.js` | 三栏渲染、事件委托、字段错误和作业展示 |
| `public/competitor-remix.css` | 页面专属布局、组件状态、编辑器和响应式样式 |

模块依赖保持单向：视图和交互模块更新 store；payload 模块只读取显式输入并返回数据；job-runner 只通过现有 API 更新作业状态；模块不直接修改彼此内部状态。

## 6. 状态模型

顶层状态包含：

- `auth`：当前用户。
- `source`：输入模式、URL、File、Object URL、Base64 缓存、元信息和读取状态。
- `selectedCapabilityId` 和 `selectedModeId`：当前功能与模式。
- `drafts`：以 `capabilityId:modeId` 为键保存参数、坐标和帧选择。
- `runs`：以本地 run ID 为键保存 provider job ID、请求快照、状态、结果和轮询信息。
- `activeRunId`：右侧当前展开的作业。

状态规则如下：

- 视频素材在所有视频功能之间共享。
- 每个功能模式独立保存草稿，切换功能或模式不清空内容。
- 更换视频时保留普通参数，清除所有依赖旧画面的坐标、帧号和预览图，并显示说明。
- 重置只恢复当前模式，不影响其他草稿、共享素材或作业。
- 作业运行不冻结草稿；用户可继续编辑并提交其他作业。
- `sessionStorage` 只保存 URL、参数、坐标、作业 ID 和非敏感请求快照。
- File、Object URL、完整 Base64 和认证信息不写入 `sessionStorage`。
- 刷新后若原素材是本地文件，页面保留草稿和作业恢复能力，但要求重新选择文件后才能再次提交。

## 7. 素材与编辑器行为

### 7.1 文件响应速度

文件拖入或选择后按以下顺序处理：

1. 同步校验文件类型和大小。
2. 立即创建 Object URL 并显示文件名、大小和视频预览。
3. 异步读取视频元信息并更新时长和尺寸。
4. 用户提交时才调用 `FileReader` 生成 API 所需的 Base64 data URL。
5. 同一 File 的 Base64 结果只读取一次；更换文件时释放旧 Object URL 和缓存。

### 7.2 坐标系统

三类编辑器统一使用 0 到 1 的归一化坐标。事件坐标必须以 `object-fit: contain` 后实际可见的视频内容矩形为基准，不以包含黑边的容器为基准。

- 点选保存 `x`、`y` 和 positive/negative 标签。
- 框选保存排序后的 `x1`、`y1`、`x2`、`y2`。
- 小于最小面积的拖动不生成有效框。
- Pointer Capture 保证拖出画布后仍能完成框选。
- 清除、撤销最后一个点和模式切换均提供明确按钮。
- 预览层不接管事件，交互面保持单一命中区域。

### 7.3 K 帧

K 帧模式以视频当前时间为主要定位方式，同时保留现有契约所需的 `frame_index`。页面沿用现有 30 fps 换算约定，并明确标注帧号为换算值；用户可通过时间、帧号和滑块定位。提交时仍发送现有 `interaction_prompt.frame_index`。

## 8. 请求和作业数据流

请求流程为：

1. capability catalog 根据功能和模式选出底层适配器。
2. payload 模块校验共享素材、模式参数和编辑器状态。
3. 本地文件输入在提交阶段完成 Base64 准备。
4. payload 模块生成与当前接口相同的 `job_type`、`input`、`options` 和 `params`。
5. job-runner 调用 `POST /api/wangzhuan/video-ops/jobs`。
6. 成功后记录 provider job ID、归档 ID 和去敏请求快照，并启动独立轮询。
7. 轮询继续使用现有查询接口；终态时停止对应作业的定时器，并在成功或待复核时读取结果。

运行中作业维持当前 3 秒查询间隔。页面不可见时暂停普通轮询，恢复可见后立即刷新；连续网络错误使用有限退避，但不改变后端作业状态。

## 9. 错误处理

- 字段错误显示在对应控件下方，并同步到右侧就绪清单。
- 文件类型、大小、读取和视频解码错误留在素材区，不清除其他草稿。
- URL 因跨域限制不能预览时允许继续提交，但明确说明画布类功能需要可预览素材。
- 编辑器未就绪时禁用相应交互，并显示需要先完成的动作。
- 提交失败保留草稿和素材；原始 Base64 不进入错误消息或日志。
- 单次轮询失败显示连接异常，不把 provider 作业改为失败。
- provider 终态失败优先展示结构化失败原因，原始诊断信息放在可展开详情。
- 取消、重试或结果读取失败只影响对应作业卡片。

## 10. 可访问性与响应式

- 所有模式、工具和命令使用原生按钮、单选、选择框或输入控件。
- 当前功能、当前模式、选中作业和禁用原因有可读状态，不只依赖颜色。
- 交互画布可聚焦，工具按钮具备明确标签；焦点环保持可见。
- 错误和作业状态更新使用适当的 `aria-live` 区域。
- 桌面使用稳定三栏网格；中等宽度缩窄导航和作业栏；移动端改为顶部功能切换和底部作业抽屉。
- 固定格式预览和工具栏使用稳定尺寸，加载、错误和动态文本不引起布局跳动。

## 11. 测试与验收

### 11.1 自动化测试

- capability mapping：全部模式准确映射到现有 10 条底层请求路径。
- payload：默认值、必填项、边界值、mask data URL 和敏感字段去除。
- store：跨模式草稿隔离、共享素材、更换素材清理坐标、多作业并行更新和恢复。
- media：拖放即时预览、提交时延迟 Base64、缓存复用和 Object URL 释放。
- editors：可见视频矩形、点坐标、框坐标、最小面积和 Pointer 流程。
- job-runner：提交、独立轮询、终态结果、网络退避、取消和通过现有接口重试原作业。
- static regression：必要 DOM 容器、模块入口和无旧节点画布依赖。

### 11.2 浏览器回归

在桌面和移动视口验证：

- 登录状态和导航可用。
- URL、点击上传和拖放输入有即时反馈。
- 5 个功能及其模式切换后草稿保持。
- K 帧点选、K 帧框选、固定区域和局部区域操作可见且坐标准确。
- 提交一个作业后仍能编辑和提交另一功能。
- 多个作业分别刷新、取消、重试和显示结果。
- 失败、空态、加载态和结果下载入口可理解且无重叠。

### 11.3 完成标准

- 页面只展示 5 个用户功能入口，底层 10 条现有执行路径均保留。
- 所有请求通过现有 `/api/wangzhuan/video-ops/jobs` 契约发送。
- 切换功能和模式不丢草稿，作业运行不锁页面。
- 本地视频选择后立即显示文件反馈和预览，不等待 Base64。
- 三类编辑器均可完成预期操作并生成正确归一化坐标。
- 自动化测试、语法检查和浏览器关键路径回归通过。

## 12. 实施边界

实施应按模块逐步替换旧页面，每一步保持可运行和可测试。旧节点画布代码、旧页面专属 CSS 和单作业全局状态只在新路径覆盖相同行为并通过回归后删除。不得顺带修改后端契约、数据库、生产配置或其他网赚页面。
